          Туториал
        </span></div> </div></div> <!----> <!----></div></div> <!----> <div data-gallery-root="" lang="ru" class="tm-article-body"><div></div> <div id="post-content-body"><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>Всем привет! В данной статье хочу показать простой пример написания Dockerfile, объяснить как это все работает, а также показать на примере использование многоэтапной сборки.</p><p>Для понимания данной статьи необходимы минимальные знания Docker, а также для использования примеров - установленный Docker Desktop локально на компьютере.</p><p>Вначале немного теории.</p><p>Dockerfile - это файл, который содержит инструкции для сборки образа. На основании образа создается и запускается контейнер.</p><p>Обратимся к официальной документации <a href="https://docs.docker.com/get-started/overview/" rel="noopener noreferrer nofollow">https://docs.docker.com/get-started/overview/</a></p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/3af/e78/0d8/3afe780d8c98a75de6abd9eaf91d0134.png" width="1009" height="686" data-src="https://habrastorage.org/getpro/habr/upload_files/3af/e78/0d8/3afe780d8c98a75de6abd9eaf91d0134.png"/><figcaption></figcaption></figure><p> Это скриншот взят с официальной страницы. </p><p>На этом скриншоте мы видим схему как создаются и запускаются контейнеры. Мы как клиенты через команды docker обращаемся к Docker daemon, который берет локальный образ (image) и на основании его запускает контейнер, если образа локально нет - то он идет в registry (это может быть docker hub) и вначале стягивает его себе на компьютер. И может кто-то спросит "А где тут во всем этим dockerfile?". Именно на основании dockerfile и создается первоначально образ.</p><p>Для тестирования написания dockerfile создадим простой проект, который по сути ничего особо делать не будет, просто будет приветствовать нас.</p><p>Зайдем на <a href="https://start.spring.io/" rel="noopener noreferrer nofollow">https://start.spring.io/</a> и создадим проект назовем его simple-dockerfile-example. Из зависимостей подключим только Spring Web.</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/eca/4ee/c5f/eca4eec5f8d5cb56c8f23f03273f342a.png" width="1684" height="947" data-src="https://habrastorage.org/getpro/habr/upload_files/eca/4ee/c5f/eca4eec5f8d5cb56c8f23f03273f342a.png"/><figcaption></figcaption></figure><p>Генерируем и открываем этот проект в IDEA.</p><p>Создаем класс ClientService:</p><pre><code class="java">@Service
public class ClientService {

    @Value("${client}")
    private String client;

    public String sayHello(){
        return "Hello!: " + client;
    }
}</code></pre><p>и класс ClientController:</p><pre><code class="java">@RestController
@RequestMapping("api/v1/client")
public class ClientController {
    private final ClientService clientService;

    @Autowired
    public ClientController(ClientService clientService) {
        this.clientService = clientService;
    }

    @GetMapping()
    public String sayHello(){
        return clientService.sayHello();
    }
}</code></pre><p>Также в application.properties пропишем одно свойство (client), то есть имя того кого будем приветствовать.</p><pre><code class="java">client=Ivan</code></pre><p>Можем протестировать наш код, запустить проект и через postman отправить get запрос на http://localhost:8080/api/v1/client  </p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/f77/ad9/7b1/f77ad97b17b4ceb692073fe2f1c63c18.png" width="937" height="446" data-src="https://habrastorage.org/getpro/habr/upload_files/f77/ad9/7b1/f77ad97b17b4ceb692073fe2f1c63c18.png"/><figcaption></figcaption></figure><p>Должны получить приветствие.</p><p>Сейчас попробуем на основании нашего проекта сделать образ и его запустить в контейнере. Для этого в корне нашего проекта создадим файл Dockerfile:</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/301/553/ac7/301553ac718f01839de67b8eb0fb6a0d.png" width="621" height="629" data-src="https://habrastorage.org/getpro/habr/upload_files/301/553/ac7/301553ac718f01839de67b8eb0fb6a0d.png"/><figcaption></figcaption></figure><p>и в данном файле пишем:</p><pre><code class="java">FROM eclipse-temurin
ARG JAR_FILE=target/*.jar
COPY ${JAR_FILE} app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "/app.jar"]</code></pre><p>Разберем что тут происходит в данном файле.</p><p>В первой строке мы указываем какой образ нужно стянуть с docker hub.</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/a65/4d3/009/a654d3009661a6ec9547cfbfe1aab4e2.png" width="1671" height="902" data-src="https://habrastorage.org/getpro/habr/upload_files/a65/4d3/009/a654d3009661a6ec9547cfbfe1aab4e2.png"/><figcaption></figcaption></figure><p>Этот образ нужен для того чтобы в контейнере была развернута своя JDK для запуска нашего проекта.</p><p>ARG JAR_FILE=target/*.jar - здесь мы создаем просто локальную переменную, которая ссылается на jar-ник нашего проекта.</p><p>COPY ${JAR_FILE} app.jar - мы копируем наш jar-ник в образ и называем его app.jar.</p><p>EXPOSE 8080 - указываем на каком внешнем порту будет доступен наш контейнер.</p><p>ENTRYPOINT ["java", "-jar", "/app.jar"] - мы запускаем наш  jar-ник.</p><p>Если кому-то не очень понятно что происходит в последней строчке, то это то же самое, что если бы мы предварительно запустили команду package и получили simple-dockerfile-example-0.0.1-SNAPSHOT.jar в папке target.</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/63c/677/4f2/63c6774f24c7147e992e909543c009d0.png" width="1871" height="814" data-src="https://habrastorage.org/getpro/habr/upload_files/63c/677/4f2/63c6774f24c7147e992e909543c009d0.png"/><figcaption></figcaption></figure><p>А затем с помощью командной строки зашли в папку target и выполним команду java -jar simple-dockerfile-example-0.0.1-SNAPSHOT.jar</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/a1d/21d/56b/a1d21d56b31649ae2a695d4016eaeb5d.png" width="1598" height="911" data-src="https://habrastorage.org/getpro/habr/upload_files/a1d/21d/56b/a1d21d56b31649ae2a695d4016eaeb5d.png"/><figcaption></figcaption></figure><p>И мы таким образом запустили наш проект. Можем сейчас зайти в postman и отправить get запрос на http://localhost:8080/api/v1/client  и получить снова Hello!: Ivan. </p><p>Останавливаем проект. Заходим в терминале в корень нашего проекта, где лежит Dockerfile и сейчас будем строить наш образ. Выполним команду docker build .</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/436/146/fed/436146fed5cc6078a97938308b9946a4.png" width="1748" height="942" data-src="https://habrastorage.org/getpro/habr/upload_files/436/146/fed/436146fed5cc6078a97938308b9946a4.png"/><figcaption></figcaption></figure><p>Docker daemon должен стянуть всё необходимое с docker hub. Наш образ готов. </p><p>Посмотреть наши образы можно через команду docker images в командной строке или использовать Docker Desktop. Находим его во вкладке Images:</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/77c/d6e/9d0/77cd6e9d0aa78327ec09e7fa04a66386.png" width="1878" height="385" data-src="https://habrastorage.org/getpro/habr/upload_files/77c/d6e/9d0/77cd6e9d0aa78327ec09e7fa04a66386.png"/><figcaption></figcaption></figure><p>Мы видим, что образ есть и он занимает 490,84 МВ. Также выполним команду docker images в командной строке.</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/929/51b/221/92951b2211a5ab0693104fe2563cb958.png" width="799" height="183" data-src="https://habrastorage.org/getpro/habr/upload_files/929/51b/221/92951b2211a5ab0693104fe2563cb958.png"/><figcaption></figcaption></figure><p>Дальше давайте запустим наш образ, то есть поместим его в контейнер.</p><p>Выполним команду docker run -p 8080:8080 b472</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/b01/587/3d3/b015873d3a3fd4c2bbb1282b013657ec.png" width="1139" height="291" data-src="https://habrastorage.org/getpro/habr/upload_files/b01/587/3d3/b015873d3a3fd4c2bbb1282b013657ec.png"/><figcaption></figcaption></figure><p>-p в данной команде мы указываем порты. Первый порт -это порт на нашем компьютере то есть наш localhost, а второй порт - это порт внутри контейнера. </p><p>b472 - это первые 4 буквы ID нашего контейнера, у Вас они будут отличаться.</p><p>После выполнения данной команды контейнер должен запуститься с нашим проектом и отправив через postman get запрос на http://localhost:8080/api/v1/client  мы снова должны получить Hello!: Ivan.</p><p>Таким образом мы создали проект, создали образ с данным проектом и развернули его через docker в контейнере.</p><p>Что плохо написано в нашем Dockerfile.</p><p>Мы не указали версию JDK (это первая строка FROM eclipse-temurin), если версия не указана, то будет скачана самая последняя доступная версия, что может привести к несовместимости нашего приложения с JDK. И во-вторых здесь мы не компилируем jar, а только его запускаем, поэтому нам бы хватило и JRE и это уменьшит размер нашего образа. Переделаем это.</p><pre><code class="java">FROM eclipse-temurin:17-jre-alpine
ARG JAR_FILE=target/*.jar
COPY ${JAR_FILE} app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "/app.jar"]</code></pre><p>Остановим наш контейнер через Docker Desktop и запустим снова команду  docker build.</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/c17/38d/3ad/c1738d3ad5c65a51e849a894d34b4634.png" width="789" height="140" data-src="https://habrastorage.org/getpro/habr/upload_files/c17/38d/3ad/c1738d3ad5c65a51e849a894d34b4634.png"/><figcaption></figcaption></figure><p>Выполним команду docker images:</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/d18/89a/ad1/d1889aad1926c5977307801f26b67a2e.png" width="784" height="110" data-src="https://habrastorage.org/getpro/habr/upload_files/d18/89a/ad1/d1889aad1926c5977307801f26b67a2e.png"/><figcaption></figcaption></figure><p>И мы увидим, что размер нового образа 188 МВ в 2,5 раза меньше, чем предыдущий, а работает все также.</p><p>Перейдем сейчас к использованию многоэтапной сборки. Суть ее в том, что как вначале чтобы сделать образ мы сами вручную запускали команду мавена и создавали jar-ник нашего проекта, а при многоэтапной сборке jar-ник будет генерироваться автоматически.</p><p>Перепишем dockerfile.</p><pre><code class="java">FROM eclipse-temurin:17-jdk-alpine as builder
WORKDIR /opt/app
COPY .mvn/ .mvn
COPY mvnw pom.xml ./
RUN ./mvnw dependency:go-offline
COPY ./src ./src
RUN ./mvnw clean install

FROM eclipse-temurin:17-jre-alpine
WORKDIR /opt/app
COPY --from=builder /opt/app/target/*.jar /opt/app/*.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "/opt/app/*.jar"]</code></pre><p>Разберемся что тут происходит. </p><p>FROM eclipse-temurin:17-jdk-alpine as builder - здесь, как мы уже знаем, указываем docker-у какой образ мы будем использовать для сборки нашего проекта и тут мы уже указываем именно jdk, так как нам понадобятся инструменты для компиляции нашего проекта. as builder - это название, которое мы присвоили, для того чтобы обратиться с другого слоя контейнера для получения данных.</p><p>WORKDIR /opt/app - создаем папки в данном слое</p><p>COPY .mvn/ .mvn - копируем папку mvn и все ее содержимое в такую же папку в корень данного слоя, для того чтобы у нас был maven.</p><p>COPY mvnw pom.xml ./ - копируем mvnw и pom.xml тоже в корень данного слоя.</p><p>RUN ./mvnw dependency:go-offline - данной строчкой мы подтягиваем все зависимости из pom.xml в наш слой, чтобы у нас в контейнере были все зависимости, необходимые для нашего проекта.</p><p>COPY ./src ./src - копируем непосредственно папку с нашим проектом.</p><p>RUN ./mvnw clean install - запускаем мавен, который все чистит и создает  jar-ник нашего проекта.</p><p>строчка 10 WORKDIR /opt/app - создаем папки в другом слое.</p><p>COPY --from=builder /opt/app/target/*<em>.jar /opt/app/*</em>.jar -
