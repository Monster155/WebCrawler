Чтобы использовать возможности взаимодействия по максимуму, требуется пройти полную авторизацию (OAuth 2.0). Но в отличие от подобных процедур авторизации, скажем, в API ВК, с Google всё несколько сложнее, особенно учитывая небольшое упущение его разработчиков. В самой документации API Google, как будто, не вполне достаточно информации для успешной полной авторизации: у меня и ряда моих коллег попытки запросов к API сервисов Google, используя полную авторизацию (файл credentials), приводили к ошибкам. Эту ошибку я демонстрирую в самом конце этой статеички, а сама статеичка -- это порядок действий в шести шагах, который позволяет избежать этой ошибки на основе обобщения моего опыта.</p><ol><li><p><a href="https://developers.google.com/youtube/v3/guides/authentication" rel="noopener noreferrer nofollow">Отсюда можно начать</a>, чтобы получить общее представление</p></li><li><p>Далее следует <a href="https://console.cloud.google.com/apis/credentials/consent" rel="noopener noreferrer nofollow">настроить consent screen</a> (To create an OAuth client ID, you must first configure your consent screen – в т.ч. создать своё приложение) </p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/a51/e1f/5ff/a51e1f5ff8b0b26f760835354f21b4f9.png" alt="Пример приложения" title="Пример приложения" width="1090" height="550" data-src="https://habrastorage.org/getpro/habr/upload_files/a51/e1f/5ff/a51e1f5ff8b0b26f760835354f21b4f9.png"/><figcaption>Пример приложения</figcaption></figure><p>Я выбрал User type: External. Scopes не менял</p></li><li><p><em>(опционально)</em> <a href="https://developers.google.com/identity/protocols/oauth2" rel="noopener noreferrer nofollow">Протоколы взаимодействия</a> пользователя, приложения и сервера </p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/612/67b/f6c/61267bf6c56297f8688e8113afd77eb0.png" alt="Пример иллюстрации протокола" title="Пример иллюстрации протокола" width="705" height="661" data-src="https://habrastorage.org/getpro/habr/upload_files/612/67b/f6c/61267bf6c56297f8688e8113afd77eb0.png"/><figcaption>Пример иллюстрации протокола</figcaption></figure></li><li><p>Создать <a href="https://console.cloud.google.com/apis/credentials/oauthclient" rel="noopener noreferrer nofollow">OAuth client ID</a>. Я создал Web client и Desktop client (могут быть привязаны к одному и тому же приложению). Получить Client ID и Client Secret, а также сохранить файл формата JSON с credentials внутри</p></li><li><p>Для проверки, что всё удалось, в API Explorer сгенерировать простейший запрос, например, как <a href="https://youtu.be/CwoxdhR1z_s" rel="noopener noreferrer nofollow">тут</a>. Любопытно, что гугловские пакеты для Python инсталируются как google-api-python-client и google-auth-oauthlib google-auth-httplib2 , а импортируются как googleapiclient и google_auth_oauthlib</p></li><li><p>Заменить в запросе строку кода <code>credentials = flow.run_console()</code> на <code>credentials = flow.run_local_server()</code> . Если всё сделать правильно, получается примерно <a href="https://drive.google.com/file/d/1xs9Iujtb1jibdPDPzt8F2G1dDxpSqDCq" rel="noopener noreferrer nofollow">так</a>.</p><p>Дальше читать не обязательно. Но если интересна причина необходимости такой замены, то рассказываю: с 2022 года происходит <a href="https://developers.google.com/identity/protocols/oauth2/resources/oob-migration" rel="noopener noreferrer nofollow">такой процесс</a>. Видимо, разработчики из Google забыли поправить в генераторе кода API Explorer. Поэтому без замены происходит следующее: если запрос написан корректно, появляется ссылка Please visit this URL to authorize this application для получения the authorization code   </p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/748/745/f3f/748745f3ff9434ff97685b1d0cc02cfa.png" alt="Если не произвести замену, то исполняется Out-of-Band (OOB) flow" title="Если не произвести замену, то исполняется Out-of-Band (OOB) flow" width="1090" height="701" data-src="https://habrastorage.org/getpro/habr/upload_files/748/745/f3f/748745f3ff9434ff97685b1d0cc02cfa.png"/><figcaption>Если не произвести замену, то исполняется Out-of-Band (OOB) flow</figcaption></figure><p>Но authorization code получить не удастся, потому что Error 400: invalid_request The out-of-band (OOB) flow has been blocked in order to keep users secure. Follow the Out-of-Band (OOB) flow migration guide linked in the developer docs below to migrate your app to an alternative method. Request details: redirect_uri=urn:ietf:wg:oauth:2.0:oob</p><figure class="full-width "><img
